## Ігролад та UI Flow

### Екран 1 — Налаштування (Setup Screen)

#### Загальна структура

При запуску додатку відображається сторінка налаштування бою: сітка 2 × 4 з картками персонажів (максимум 8 слотів).

```
┌──────────┬──────────┐
│  Char 1  │  Char 2  │  ← активні за замовчуванням
│ (гравець)│  (ком'р) │
├──────────┼──────────┤
│  [+ Add] │  [+ Add] │
├──────────┼──────────┤
│  [+ Add] │  [+ Add] │
├──────────┼──────────┤
│  [+ Add] │  [+ Add] │
└──────────┴──────────┘
           [ГОТОВО]
```

- Кнопка `[+ Add]` активує слот і додає нового персонажа з випадковими статами.
- Активний слот можна деактивувати (видалити персонажа), якщо живих персонажів залишається ≥ 2.
- Кнопка **ГОТОВО** стає активною лише якщо активних персонажів ≥ 2.

#### Налаштування кожного персонажа

| Поле | Тип | Значення за замовч. | Обмеження |
|------|-----|---------------------|-----------|
| Сила (S) | number input | випадково | ≥ 1; S + A + I = 100 |
| Спритність (A) | number input | випадково | ≥ 1; S + A + I = 100 |
| Інтелект (I) | number input | випадково | ≥ 1; S + A + I = 100 |
| Тип зброї | select (6 опцій) | випадково | — |
| Керування | toggle Player / AI | Char 1 → Player, решта → AI | — |
| Команда | color picker (8 кольорів) | Char 1 → Red, Char 2 → Blue, ... | кожен колір унікальний |

#### Генерація випадкових статів (сума = 100)

```typescript
function randomStats(total = 100): CharacterStats {
  // Два різних розрізи в [1, total-1] — гарантує S,A,I >= 1
  const positions = new Set<number>();
  while (positions.size < 2) {
    positions.add(1 + Math.floor(Math.random() * (total - 1)));
  }
  const [p1, p2] = [...positions].sort((a, b) => a - b);
  return { S: p1, A: p2 - p1, I: total - p2 };
}
```

#### Типи зброї

| # | Формула D | Назва (приклад) | Вартість F |
|---|-----------|-----------------|-----------|
| 1 | `S + S` | Дворучний меч | 3 |
| 2 | `A + A` | Кинджали | 2 |
| 3 | `I + I` | Посох | 3 |
| 4 | `S + A` | Одноручний меч | 2 |
| 5 | `A + I` | Рапіра | 2 |
| 6 | `I + S` | Бойовий молот | 3 |

> Вартість F є базовою і може балансуватись. Зброї з однією парою однакових статів коштують дорожче, оскільки дають більший потенціал шкоди.

#### Модель даних налаштувань

```typescript
type WeaponType = 'SS' | 'AA' | 'II' | 'SA' | 'AI' | 'IS';
type Controller = 'player' | 'ai';

const TEAM_COLORS = {
  1: '#e74c3c', // Red
  2: '#3498db', // Blue
  3: '#2ecc71', // Green
  4: '#f1c40f', // Yellow
  5: '#9b59b6', // Purple
  6: '#e67e22', // Orange
  7: '#ff69b4', // Pink
  8: '#ecf0f1', // White
} as const;
type TeamColor = keyof typeof TEAM_COLORS;

interface WeaponConfig {
  type: WeaponType;
  stat1: 'S' | 'A' | 'I';
  stat2: 'S' | 'A' | 'I';
  fatigueCost: number;
}

interface CharacterSetup {
  id: string;
  stats: CharacterStats;       // S + A + I = 100
  weapon: WeaponConfig;
  controller: Controller;
  team: TeamColor;
}

interface SetupState {
  slots: Array<CharacterSetup | null>; // довжина завжди 8; null = неактивний слот
}
```

#### Валідація перед стартом

```
S >= 1, A >= 1, I >= 1
S + A + I === 100
Активних персонажів: 2..8
Кожен активний персонаж має команду та тип зброї
```

### Екран 2 — Арена (Battle Screen)

#### Стартове розміщення персонажів

Усі активні персонажі розміщуються на гексагональному полі рівномірно — по колу радіусом R навколо центру поля.

```
Кут для персонажа i (з N персонажів):
  angle_i = (2π / N) × i

Стартова позиція (у cube coords):
  Перевести полярні (R, angle_i) у hex-координати та округлити до найближчого гекса.
```

Радіус R (відстань від центру до стартових позицій) відповідає таблиці з розділу "Розмір поля". Для N=2 це R=3, що дає 3 вільних гекси між персонажами.

Якщо два персонажі потрапляють на один гекс після округлення — зсунути одного на сусідній вільний гекс за годинниковою стрілкою.

#### Відображення арени

- Гексагональна сітка (pointy-top), кожен персонаж — токен кольору своєї команди.
- Доступні для руху гекси підсвічуються при виборі дії "Рух".
- Доступні для атаки гекси підсвічуються при виборі дії "Атака" (суміжні, дальність 1).

##### Панель черги (зверху екрана)

Горизонтальний ряд карток усіх живих персонажів, відсортованих зліва направо за поточною чергою ходів (від першого до останнього у раунді).

Кожна картка містить:
- Колір команди (фон або рамка)
- Номер персонажа (Char 1..8)
- Поточні стати: `S` / `A` / `I`
- Значення ініціативи поточного раунду
- Іконка зброї

Картка **активного** персонажа (чий зараз хід) виділена візуально: збільшена, яскравіша рамка або підсвітка. Картки вже відіграних у цьому раунді — приглушені. Картки персонажів, що вибули (мертві), зникають із черги.

```
┌──────┐  ┌══════╗  ┌──────┐  ┌──────┐
│ Init │  ║ Init ║  │ Init │  │ Init │
│  8   │  ║  12  ║  │  5   │  │  3   │
│ S:14 │  ║ S:20 ║  │ S: 8 │  │ S: 6 │
│ A: 9 │  ║ A:15 ║  │ A: 7 │  │ A: 4 │
│ I: 7 │  ║ I:10 ║  │ I: 5 │  │ I: 3 │
│ 🗡️   │  ║ 🪄   ║  │ 🗡️   │  │ 🏹   │
└──────┘  └══════╝  └──────┘  └──────┘
 відіграв   ← АКТИВНИЙ →   ще не ходив
```

#### Цільова платформа

Desktop-only. Базовий розмір canvas: **1280 × 720** (16:9). Phaser `Scale.FIT` — масштабується під розмір вікна без зміни пропорцій. Mobile — можливе розширення у майбутньому.

#### Перехід Setup → Battle

```
[Натиснуто ГОТОВО]
  1. Валідація SetupState
  2. Перетворити CharacterSetup[] → CharacterState[] (F=0, KnockedOut=false)
  3. Розрахувати стартові hex-позиції
  4. Перейти до BattleScene, передати список персонажів та позиції
  5. Запустити Раунд 1
```

## 1. Характеристики персонажа

### Модель даних

```typescript
interface CharacterStats {
  S: number; // Сила    — мін. 1
  A: number; // Спритність — мін. 1
  I: number; // Інтелект  — мін. 1
}

interface CharacterState {
  stats: CharacterStats;
  fatigue: number;        // поточна Втома F
  knockedOut: boolean;    // true → наступний хід без захисту
  position: HexCoord;     // поточна позиція на полі { q, r, s }
  initOrder: number;      // порядковий номер при ініціалізації (для розв'язку рівності)
}
```

### Похідні значення

```
HP_max = S + A + I
E_max  = I          // поточне значення, змінюється при отриманні шкоди
```

### Умова смерті

Персонаж гине, якщо **за один удар** `NetD ≥ HP − 2` (тобто залишковий HP після удару ≤ 2).

Якщо `NetD < HP − 2` — шкода розподіляється пропорційно (кожен стат не падає нижче `1`). Мінімальний HP через пропорційний розподіл = 3 (усі стати = 1), тому вбивчий удар завжди перевіряється явно до розподілу.

## 2. Система бою

### Функція Random

```
Random(X) = рівномірне ціле з [max(1, floor(X/5)), X] включно
```

Мінімум завжди ≥ 1 і складає ≈20% від X. Приклади: X=10 → [2,10]; X=6 → [1,6]; X=1 → [1,1].

### Формула атаки

```
D    = Random(Stat1 + Stat2)   // залежить від типу зброї
Def  = Random(S)               // замінюється на 0 якщо KnockedOut = true
NetD = max(0, D - Def)
```

#### Типи зброї, статі та дальність

| # | Тип | Формула D | Дальність |
|---|-----|-----------|-----------|
| 1 | Дворучний меч | `S + S` | 1 гекс |
| 2 | Кинджали | `A + A` | 1 гекс |
| 3 | Посох | `I + I` | 1 гекс |
| 4 | Одноручний меч | `S + A` | 1 гекс |
| 5 | Рапіра | `A + I` | 1 гекс |
| 6 | Бойовий молот | `I + S` | 1 гекс |

Уся зброя наразі має дальність атаки **1** — тобто лише суміжні гекси. Дружнього вогню немає: атака на союзника неможлива.

### Пропорційний розподіл шкоди

```
apply = min(NetD, (S-1) + (A-1) + (I-1))   // фактично застосована шкода

ΔS = floor(apply × S / HP)
ΔA = floor(apply × A / HP)
ΔI = floor(apply × I / HP)
```

**Правило мінімуму:** жоден стат не може впасти нижче `1`, поки хоча б один інший > `1`.

- Фактично застосовуване пошкодження обмежене: `apply = min(NetD, (S-1)+(A-1)+(I-1))`.
- Надлишок `NetD - apply` просто поглинається (не переноситься).

**Приклад:** S=100, A=50, I=50, HP=200, NetD=120.
- Частки: S=50%, A=25%, I=25%
- ΔS = 120×0.5 = 60, ΔA = 120×0.25 = 30, ΔI = 120×0.25 = 30
- Результат: S=40, A=20, I=20 ✓

**Смерть:** якщо `NetD ≥ HP − 2` (залишковий HP ≤ 2) → персонаж гине, перевіряється **до** розподілу.

**Округлення дробів:** спочатку за спаданням дробової частини (`floor`-залишок), потім — жадібне заповнення залишкової ємності (для випадків, де кеп обмежує один прохід).

```typescript
function distributeNetDamage(
  netD: number, s: number, a: number, i: number
): { S: number; A: number; I: number; dead: boolean } {
  if (netD <= 0) return { S: 0, A: 0, I: 0, dead: false };

  const hp = s + a + i;
  if (netD >= hp - 2) return { S: s, A: a, I: i, dead: true }; // вбивчий удар (залишковий HP ≤ 2)
  const cap = { S: s - 1, A: a - 1, I: i - 1 };
  // apply = netD завжди (смертельна перевірка вище гарантує netD ≤ hp-3 = сума caps)
  const ideal = { S: netD * s / hp, A: netD * a / hp, I: netD * i / hp };
  const d = {
    S: Math.min(cap.S, Math.floor(ideal.S)),
    A: Math.min(cap.A, Math.floor(ideal.A)),
    I: Math.min(cap.I, Math.floor(ideal.I)),
  };
  let rem = netD - d.S - d.A - d.I;
  // Прохід 1: розподіл залишку за спаданням дробової частини
  for (const k of (['S', 'A', 'I'] as const).sort((x, y) => (ideal[y] % 1) - (ideal[x] % 1))) {
    if (rem <= 0) break;
    if (d[k] < cap[k]) { d[k]++; rem--; }
  }
  // Прохід 2: жадібне заповнення (для випадків коли прохід 1 не вичерпав rem)
  for (const k of ['S', 'A', 'I'] as const) {
    if (rem <= 0) break;
    const avail = cap[k] - d[k];
    const add = Math.min(rem, avail);
    d[k] += add; rem -= add;
  }
  return { ...d, dead: false };
}
```

## 3. Ініціатива та черговість ходів

```
Initiative(char) = Random(A)
```

На початку кожного раунду кожен живий персонаж кидає ініціативу. Хід виконується від **більшого** до **меншого** значення.

**Вирішення рівності (каскадно):**
1. Більша поточна `A`
2. Більша сума поточних `S + A + I`
3. Менший порядковий номер при ініціалізації (хто був доданий раніше в Setup)

### Структура раунду

```
[Раунд]
  для кожного персонажа → кинути Initiative
  відсортувати за спаданням
  для кожного персонажа в черзі:
    → [Хід персонажа]
    → перевірити смерті
  кінець раунду
```

## 4. Пересування на гексагональному полі

### Визначення Еталона

| Ситуація | Еталон |
|----------|--------|
| Група (> 2 осіб) | Персонаж з `A`, найближчою до `Ā` **у більшу сторону** |
| Дуель (2 особи) | Персонаж з **меншою** `A` |

```
Ā = середнє значення A усіх живих персонажів у групі
```

> Еталон і кількість кроків **перераховуються щоразу на початку кожного раунду** — разом із кидком ініціативи. Загибель персонажів між раундами змінює середнє `Ā`, отже Еталон і кроки кожного живого персонажа можуть змінитися.

### Розрахунок кроків

```
BaseRef = 2   // Еталон завжди отримує 2 кроки

Для кожного персонажа p:
  ratio = p.A / Ref.A

  якщо ratio < 0.7  → Steps = 1
  інакше             → Steps = 2 + floor((ratio - 1) / 0.5)

  Steps = clamp(Steps, 1, 5)
```

#### Таблиця прикладів (Ref.A = 10)

| A персонажа | ratio | Steps |
|-------------|-------|-------|
| 6 | 0.60 | 1 |
| 8 | 0.80 | 2 |
| 10 | 1.00 | 2 |
| 15 | 1.50 | 3 |
| 20 | 2.00 | 4 |
| 25 | 2.50 | 5 |

### Гексагональна сітка

Орієнтація: **pointy-top** (гекс гострим кутом догори).

Система координат: **Cube coordinates** `(q, r, s)` де `q + r + s = 0`.

- Відстань між гексами: `(|Δq| + |Δr| + |Δs|) / 2`
- Бібліотека: [redblobgames hex grid](https://www.redblobgames.com/grids/hexagons/) або вбудований Phaser 3 `Hexagonal Tilemap`

### Розмір поля

Поле має форму правильного гексагона (hex grid radius). Адаптується під кількість персонажів:

| N персонажів | Радіус поля (гексів) | Стартовий радіус R |
|:---:|:---:|:---:|
| 2 | 3 | 3 (3 вільних гекси між ними) |
| 3 | 4 | 4 |
| 4 | 4 | 4 |
| 5 | 5 | 5 |
| 6 | 5 | 5 |
| 7 | 6 | 6 |
| 8 | 6 | 6 |

> Для 2 персонажів: діаметр поля = 7, стартова відстань між ними = 3 вільних гекси (тобто R=3). Кожен наступний персонаж понад 2 додає +1 до радіусу поля.

Переміщення **крізь союзників заборонено** — зайняті союзником гекси є непрохідними (BFS не проходить через них).

## 5. Система енергії та втоми

### Параметри

```
E_max = I   // динамічний: змінюється разом зі статом I
F     = 0   // поточна Втома; скидається до 0 на початку ходу
```

### Вартість дій

| Дія | Вартість F |
|-----|-----------|
| 1 крок пересування | +1 |
| Фізична атака | +2 |
| Атака зброєю | вартість F зброї (з таблиці типів) |

### Стан непритомності

```
якщо F >= E_max (у будь-який момент під час ходу):
  → хід негайно завершується
  → прапор KnockedOut = true
  → до початку наступного ходу: Def = 0 (ігнорує кидок)

на початку наступного ходу:
  → KnockedOut = false
  → F = 0
```

### Взаємозв'язок шкоди та системи втоми

Оскільки `E_max = I`, і `I` зменшується пропорційно при отриманні шкоди:
- Збитий `I` → менший `E_max` → персонаж швидше непритомніє
- Збита `A` → менше кроків + менша ініціатива
- Збита `S` → слабший захист + менша шкода важкою зброєю

## 6. Цикл гри (Core Loop)

```
[Початок бою]
  ініціалізувати всіх персонажів
  F = 0 для всіх

[Раунд]
  1. Кидок ініціативи: Initiative = Random(A) для кожного
  2. Сортування за спаданням Initiative → черга ходів
  3. Для кожного персонажа в черзі:
     a. Скинути F = 0; зняти KnockedOut; перерахувати Steps
     b. Гравець (або AI) вибирає дії:
        — переміщення до атаки (0..Steps кроків)
        — атака (max 1 за хід, лише ворог на суміжному гексі)
        — переміщення після атаки (залишок кроків)
        — або будь-яка підмножина цих дій
     c. Після кожної дії: F += вартість; перевірити F >= E_max
     d. Якщо F >= E_max → KnockedOut = true, завершити хід
     e. Після атаки: розрахувати NetD, розподілити шкоду по статах
     f. Перевірити умову смерті: NetD ≥ HP цілі − 2 → dead
  4. Перевірити умову завершення бою:
     жива лише одна команда → вона переможець
  5. Наступний раунд

[Кінець бою]
  Переможець — остання жива команда (1 або кілька персонажів).
  Нічия неможлива: завжди залишається хоча б один персонаж.
```

## 8. Ключові інваріанти для тестів

```typescript
// 1. Якщо не вбивчий удар — вся шкода NetD розподілена повністю
assert(dead || ΔS + ΔA + ΔI === netD)

// 2. Жоден стат не падає нижче 1 (якщо удар не вбивчий)
assert(dead || (S - ΔS >= 1 && A - ΔA >= 1 && I - ΔI >= 1))

// 3. Смерть тільки якщо netD >= HP - 2 (залишковий HP ≤ 2)
assert(!dead || netD >= S + A + I - 2)

// 4. Random(X) завжди в [max(1, floor(X/5)), X]
assert(val >= Math.max(1, Math.floor(X / 5)) && val <= X)

// 5. Кроки завжди в межах [1, 5]
assert(steps >= 1 && steps <= 5)

// 6. При KnockedOut Def = 0 (незалежно від кидка)
assert(knocked ? def === 0 : true)

// 7. E_max завжди дорівнює поточному I
assert(eMax === character.stats.I)

// 8. randomStats: S+A+I = 100, кожен >= 1
const { S, A, I } = randomStats(); assert(S+A+I === 100 && S>=1 && A>=1 && I>=1)
```
